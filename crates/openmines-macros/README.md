# Руководство по декларативной разметке `gui!` в OpenMines

Декларативный макрос `gui!` (`openmines-macros`) позволяет собирать и валидировать GUI-окна для legacy-клиента игры на этапе компиляции, используя HTML/XML-подобный синтаксис (по аналогии с JSX в React/Solid).

---

## Базовый синтаксис

Вся разметка оборачивается в корневой тег `<window>`:

```rust
use crate::net::session::ui::horb::gui;

let name = "Alice";
let is_admin = true;

let response_json = gui! {
    <window title="Личный кабинет" style="canv-w=100;canv-h=100">
        <text>"Добро пожаловать, " {name} "!"</text>
    </window>
}.to_json();
```

### Переменные и выражения Rust
Внутри тегов и атрибутов вы можете использовать стандартный синтаксис Rust, оборачивая переменные в фигурные скобки `{}`:
* Атрибуты: `active={is_admin}` или просто `active=is_admin`.
* Текстовый контент: `<text>"Привет, " {name}</text>`.

---

## Доступные компоненты

### 1. Корневое окно (`<window>`)
* **Атрибуты:**
  * `title="..."` (обязательный) — заголовок окна.
  * `style="..."` (опциональный) — CSS-подобные стили окна.

### 2. Вкладки (`<tabs>`)
Внутри `<tabs>` разрешены только элементы `<tab>`.
* **Атрибуты `<tab>»:**
  * `label="..."` (обязательный) — текст вкладки.
  * `action="..."` (опциональный) — пакет-команда, отправляемая серверу при клике.
  * `active={bool}` (опциональный) — активна ли вкладка (по умолчанию `false`).

```xml
<tabs>
    <tab label="Инфо" active=true />
    <tab label="Магазин" action="open_shop" />
</tabs>
```

### 3. Интерактивные формы (`<form>»)
Предназначены для ввода данных, настроек или вывода структурированной информации.
Внутри формы поддерживаются следующие строки:

* **`<text-row label="..." />`** — простая строка текста (подпись).
* **`<toggle-row label="..." key="..." active={bool} />`** — интерактивный переключатель (флажок).
* **`<uint-row label="..." key="..." value={number} />`** — поле ввода целого неотрицательного числа.
* **`<button-row label="..." btn-label="..." action="..." />`** — текстовая строка с кнопкой действия справа.
* **`<dropdown-row label="..." key="..." selected={index}>`** — выпадающий список. Дочерними элементами могут быть только `<option>` с атрибутами `value` и `label`.

```xml
<form>
    <text-row label="Личные настройки" />
    <toggle-row label="Включить авто-добычу" key="autodig" active=true />
    <dropdown-row label="Выбор режима" key="mode" selected=0>
        <option value=0 label="Мирный" />
        <option value=1 label="Агрессивный" />
    </dropdown-row>
</form>
```

### 4. Кнопки действий (`<buttons>`)
Группа кнопок в нижней части окна. Внутри разрешены только `<button>`.
* **Атрибуты `<button>»:**
  * `label="..."` (обязательный) — текст кнопки.
  * `action="..."` (обязательный) — экшен-команда.

```xml
<buttons>
    <button label="Сохранить" action="save_settings" />
    <button label="Закрыть" action="exit" />
</buttons>
```

### 5. Холст для рисования (`<canvas>`)
Позволяет рисовать карты, графику или расставлять точки телепортации. Внутри разрешены только графические примитивы: `<rect>` и `<teleport-point>`.
* **Атрибуты `<canvas>»:**
  * `style="..."` — размеры холста (например, `canv-w=100;canv-h=100`).
* **Элемент `<rect>` (прямоугольник):**
  * `x`, `y`, `w`, `h` (обязательные) — координаты и размеры в пикселях.
  * `color="..."` (обязательный) — HEX-код цвета без знака `#` (например, `ff0000`).
* **Элемент `<teleport-point>` (маркер телепорта):**
  * `x`, `y` (обязательные) — координаты.
  * `action="..."` (обязательный) — экшен-команда при клике.

```xml
<canvas style="canv-w=50;canv-h=50">
    <rect x=10 y=10 w=18 h=18 color="ff0000" />
    <teleport-point x=20 y=20 action="tp:base" />
</canvas>
```

## Строгая валидация при компиляции

Компилятор Rust автоматически отклонит сборку с понятными сообщениями об ошибках, если вы:
1. Забудете обязательный атрибут (например, `title` в `<window>` или `key` в `<toggle-row>`).
2. Попытаетесь вставить неразрешенный тег внутрь контейнера (например, `<rect>` внутрь `<form>` или `<div />` внутрь `<tabs>`).
3. Сделаете опечатку в названии тега или дочернего элемента.
