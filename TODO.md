# Сравнение программатора: Rust-сервер vs C# референс-сервер

Проведен детальный построчный аудит серверной реализации программатора на **Rust-сервере** ([programmator.rs](file:///Users/murasama/Projects/games/OpenMines/server/game/programmator.rs)) в сравнении с оригинальным **C# сервером** ([ProgrammatorData.cs](file:///Users/murasama/Projects/games/OpenMines/server_reference/GameShit/Programmator/ProgrammatorData.cs)).

---

## 1. Общие результаты аудита

Реализация программатора на Rust-сервере выполнена **практически 1-в-1** с C# референс-сервером. Логика разбора LZMA+Base64-программ, вызовы подфункций, передача смещений (`startoffset`), логика булевых операторов (`And`/`Or`), а также поведение при респавне игрока (`OnDeath`) портированы дословно.

Однако обнаружен ряд мелких архитектурных уточнений и расширений.

## 2. Различия в байт-кодах (ID команд)

Rust-сервер полностью поддерживает весь набор кодов оригинального C# сервера, но дополнительно содержит несколько **расширений**, которые на C# сервере не распознаются (маппятся в `ActionType.None`):

| Код | Команда в Rust | Поведение в Rust | Статус в C# сервере |
| :---: | :--- | :--- | :--- |
| **98** | `ActionType::IsSlime` | Проверяет, является ли клетка слизью/кислотой (ячейки `66, 67, 68, 82, 86, 95, 96, 118`) | Отсутствует |
| **106** | `ActionType::IsInGun` | Заглушка проверки пушки (всегда `false`) | Отсутствует |
| **162** | `ActionType::HandModeOn` | Включает режим `hand_mode_active = true` (программа не блокирует ручной ход) | Отсутствует |
| **163** | `ActionType::HandModeOff` | Выключает полуручной режим | Отсутствует |
| **164** | `ActionType::MacrosGun` | Посылает команду зарядки пушки в `ProgrammatorAction::FillGun` | Отсутствует |
| **165** | `ActionType::MacrosDigAround` | Сканирует 3 клетки (впереди, слева, справа) на кристаллы и копает первый найденный | Отсутствует |

> [!NOTE]
> В C# клиенте (Unity) для полуручного режима используются коды `179` (`HANDMODE_ON`) и `180` (`HANDMODE_OFF`). В текущем Rust-сервере (как и в C# сервере) эти коды маппятся в `ActionType::None`.

## 3. Различия в реализации `ActionType::IsRoad`

* **C# сервер:** Проверяет, является ли клетка дорогой на **основном слое** мира:

    ```csharp
    Check(p, (x, y) => World.isRoad(World.GetCell(x,y)));
    ```

* **Rust-сервер:** Проверяет наличие дороги на **специальном mmap-слое дорог** (`road.mapb`):

    ```rust
    check_cell(&mut *prog, pos, world, |x, y, w| w.get_road_cell(x, y) != 0);
    ```

    *Это обусловлено многослойной архитектурой мира в Rust-версии.*

## 4. Корректность переходов при `GoTo` (Исправление бага C#)

В оригинальном C# коде при переходе по несуществующей метке в блоке `GoTo` содержится логическая ошибка (баг):

```csharp
else
{
    cFunction = startpoint.name;
    current.current = startpoint.pos; // current ссылается на СТАРУЮ функцию, меняя её указатель вместо новой
}
```

* **В Rust-сервере** эта ошибка исправлена: переход корректно сменяет функцию на стартовую и выставляет позицию в структуре новой функции.
