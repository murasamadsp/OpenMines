# World Physics Backlog

Дата: 2026-07-09.

## Сыпучка

Текущий `granular_physics_system` работает вокруг ECS-сущностей игроков. Это
достаточно для локального smoke, но неверно для игры: сыпучка является частью
глобальной симуляции мира и должна продолжать работать синхронно на карте, а не
только в радиусе игрока.

Целевая модель:

- одна система для всей сыпучки: обычная `falltype=sand` (песок, слизь, магма)
  и валуны `boulder`;
- детерминированный порядок обработки чанков и клеток;
- активные чанки/frontier вместо полного скана всей mmap-карты каждый тик;
- dirty/neighbor propagation: если в чанке была сыпучка или движение на границе,
  соседние чанки получают шанс продолжить физику;
- бюджет работы на тик с переносом остатка, чтобы не фризить игровой цикл;
- HB/broadcast только для реально изменённых клеток и только видимым клиентам;
- профиль в `tickprof`: active chunks, scanned cells, moved cells, skipped budget,
  frontier size, dirty chunks.

Запрещённое решение: сканировать всю карту целиком на каждом physics tick. При
текущем мире `1024x18016` это миллионы клеток и гарантированный лаг.

Открытый вопрос: точная синхронизация с C#/JS reference. В C# есть
`shouldbeloaded/updlasttick`; в Rust нужна эквивалентная семантика на чанках,
но без привязки только к онлайн-игрокам.
