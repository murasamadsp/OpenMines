## 3. Как строятся паки (футпринт)

Футпринт каждого пака — data-driven из `buildings.json` → массив `cells`
(`{dx, dy, type}`). При установке `place_pack_cells` кладёт эти клетки в мир;
при сносе `clear_pack_cells` чистит. GUI открывается, когда игрок **стоит на
проходимой клетке футпринта** (`packs.rs:handle_pack_action` →
`building_cells().any(|cell| pack.x+dx==px && pack.y+dy==py)`).

Типы клеток (из `cells.json`) и их роль в схеме user'а
(`X` = каркас/угол, `0` = вход, `j` = GUI):

| type | имя (cells.json) | проходима | роль |
|------|------------------|-----------|------|
| 106 | Невидимый неразрушаемый блок | нет | каркас/стена — `X` |
| 38 | Угол здания | нет (fall 1000) | угол — `X` |
| 37 | Дверь в здании | да (isEmpty) | GUI-клетка — `j` |
| 32 | Фон | да (place_over) | проход/вход — `0` |
| 35 | Фон с дорогой | да | проход с дорогой |
| 30 | Ворота | да (isEmpty) | клетка Gate |

Пример (Market 5×5, числа = type): углы 38, стены 106, центральный
крест дверей 37 (по ним открывается GUI):
```
38 106 37 106 38
106 106 37 106 106
37  37 37  37  37
106 106 37 106 106
38 106 37 106 38
```

Чтобы добавить новый пак: (1) запись в `buildings.json` (code/cost/charge/hp +
`cells`), (2) вариант в `PackType` + `config_json_key`/`from_str`, (3) кейс
установки (предмет в `heal_inventory.rs` и/или кнопка меню в `social/buildings.rs`),
(4) GUI-хендлер в `gui_buttons.rs` (если есть окно).
