### ❓ Клиент рисует, но на сервере НЕТ даже enum-типа

**Визуальное опознание (2026-06-23).** Символы есть в `PackType` C# не значатся
(полный enum: `T R G M U L F Q O W J Y D` + `None`), но клиент `PackSprite.cs`
рендерит их реальными спрайтами из `Resources/packs/pack_sheet.png`. Маппинг
`num → pack_sheet_<num>` подтверждён калибровкой (5=Market-вывеска, 1=↑UP,
3=R↻Resp, 11=Gun-турель, 31=jobs — совпали). Спрайты вырезаны по rect из `.meta`
и осмотрены. **Реализовывать НУЖНО** (решение user) — функцию подтвердить по
оригиналу.

| Символ | sprite # | Что изображено (по спрайту) | Вероятная функция |
|--------|----------|------------------------------|-------------------|
| C | -1 | спрайта нет (`num=-1`, не ставится) | внутренний/маркерный пак без рендера |
| B | 6 | жёлтая гранёная сфера, красно-белое ядро, **анимир.** | энергоядро / мина / реактор |
| b | 8 | жёлтый каркасный глобус (сетка-сфера) | глобус/радар — кандидат на **Science** (класс есть в js без char) |
| P | 26 | красный круг с белым флажком (спрайт общий с `D`) | клановый флаг-маркер (как FLAGBLYAT `D`) |
| Z | 33 | тёмный хаб с кольцом фиолетовых самоцветов | портал/телепорт-вариант или самоцветный реактор |
| c | 9 | вывеска «CLANS» со сферой меж стоек | клановое здание (штаб/доска кланов) |
| I | — | кейса в `PackSprite` нет → дефолт `num=0` | не самостоятельный спрайт |

NB: одиночные `b`/`c` ещё и ключи **кристаллов** в cargo (js: `b`синь `c`голь),
но это иной namespace — как пак-типы они выше. Полный кристальный набор cargo:
`g`зель `c`голь `r`крась `b`синь `v`фиол `w`бель.

Дальше нужна твоя сверка с оригиналом: что КАЖДЫЙ пак делает (GUI/механика),
чтобы портировать. js-класс `Science` (footprint-сетка есть, char не назначен) —
сильный кандидат для `b`/`B`.

## 3. Как строятся паки (футпринт)

Футпринт каждого пака — data-driven из `buildings.json` → массив `cells`
(`{dx, dy, type}`). При установке `place_pack_cells` кладёт эти клетки в мир;
при сносе `clear_pack_cells` чистит. GUI открывается, когда игрок **стоит на
проходимой клетке футпринта** (`packs.rs:handle_pack_action` →
`building_cells().any(|cell| pack.x+dx==px && pack.y+dy==py)`).

Типы клеток (из `cells.json`) и их роль в схеме user'а
(`X` = каркас/угол, `0` = вход, `j` = GUI):

| type | имя (cells.json) | проходима | роль |
|------|------------------|-----------|------|
| 106 | Невидимый неразрушаемый блок | нет | каркас/стена — `X` |
| 38 | Угол здания | нет (fall 1000) | угол — `X` |
| 37 | Дверь в здании | да (isEmpty) | GUI-клетка — `j` |
| 32 | Фон | да (place_over) | проход/вход — `0` |
| 35 | Фон с дорогой | да | проход с дорогой |
| 30 | Ворота | да (isEmpty) | клетка Gate |

Пример (Market 5×5, числа = type): углы 38, стены 106, центральный
крест дверей 37 (по ним открывается GUI):
```
38 106 37 106 38
106 106 37 106 106
37  37 37  37  37
106 106 37 106 106
38 106 37 106 38
```

Чтобы добавить новый пак: (1) запись в `buildings.json` (code/cost/charge/hp +
`cells`), (2) вариант в `PackType` + `config_json_key`/`from_str`, (3) кейс
установки (предмет в `heal_inventory.rs` и/или кнопка меню в `social/buildings.rs`),
(4) GUI-хендлер в `gui_buttons.rs` (если есть окно).
