# Отладка и диагностика (OpenMines)

В проекте используется связка **CodeLLDB** + **VS Code** + **Cargo Workspace** для быстрой и детальной отладки Rust-кода. 

---

## 1. Профили запуска отладки (VS Code `.vscode/launch.json`)

Все дебаг-профили используют интерактивный терминал (`"console": "integratedTerminal"`). Это позволяет:
* Использовать клавиатуру для ввода команд в консоль администратора.
* Сохранять ANSI-раскраску логов в терминале для лучшей читаемости.

### Доступные профили:
*   **`Rust: Launch Server (Debug)`** — стандартный запуск сервера с логами уровня `debug`. Базовый профиль для работы.
*   **`Rust: Launch Server (Verbose Trace)`** — запуск с логами уровня `trace`. Будет логгировать каждый SQL-запрос, пересылаемые байты сетевого протокола, состояние планировщика Bevy ECS и блокировки.
*   **`Rust: Launch Server with Regen World`** — запускает сервер с автоматическим сбросом и генерацией мира заново. Удаляет старую базу данных SQLite и регенерирует файлы карты `.map` на основе шумов Перлина.
*   **`Rust: Launch Server on Port 8095 (Proxy Backend)`** — запускает сервер на порту 8095. Полезно при разработке/проверке механизма прокси для горячей замены бэка без отключения клиентов.
*   **`Rust: Debug Tests`** — компилирует и отлаживает тестовый таргет (тесты в кодовой базе).
*   **`Rust: Launch Proxy (Debug)`** — запускает TCP-прокси для ротации бэкендов.
*   **`Rust: Launch Loadtest (Debug)`** — запускает утилиту нагрузочного тестирования на 500 ботов.

## 2. Интерактивная консоль администратора (CLI)

После успешного запуска сервера в терминале отладки инициализируется интерактивная консоль.
> [!NOTE]
> Все интерактивные команды должны начинаться со знака вопроса `?`.

### Основные команды CLI:
*   `?help` — показать список всех доступных команд.
*   `?stats` — вывести сводную игровую статистику (онлайн, количество активных сущностей, тики).
*   `?kick <id>` — принудительно отключить игрока по ID.
*   `?save` — выполнить форсированный сброс (flush) игрового мира из памяти на диск и записать журнал транзакций.
*   `?shutdown` — корректно завершить работу сервера с сохранением всех состояний.

## 3. Фильтрация логов через `RUST_LOG`

Сервер использует систему логирования `tracing` с динамическим фильтром. Настройки фильтра считываются из переменной окружения `RUST_LOG` (или `M3R_LOG`).

### Полезные фильтры:
*   `RUST_LOG=debug` — стандартный режим разработчика (показывает события сессий и базовая информация).
*   `RUST_LOG=info,sqlx=warn,bevy_ecs=warn` — скрывает шумные логи библиотеки БД и Bevy ECS, оставляя только игровую логику.
*   `RUST_LOG=openmines_server::net::session::connection=trace` — показывает побайтовый трейс разбора сетевых пакетов только для модуля подключений.

## 4. Особенности отладки асинхронного Rust

Отладка асинхронного кода в Rust (Tokio) имеет особенности, так как компилятор преобразует async-функции в конечные автоматы, а выполнение переключается между потоками планировщика (thread pool).

### Рекомендации:
1.  **Точки останова на `.await`:** 
    Не пытайтесь использовать команду «Step Over» (F10) на строках с `.await`. Это приведёт к переходу отладчика в недра планировщика Tokio. 
    **Вместо этого ставьте точку останова (breakpoint) на следующую строчку кода после `.await`** и нажимайте кнопку продолжения «Continue» (F5).
2.  **Проверка значений переменных:**
    Благодаря включенной опции `lldb.evaluateForHovers` вы можете навести курсор на любую переменную типа `Arc<T>`, `Mutex<T>` или `Option<T>` для просмотра её внутренней структуры без ручного разыменования.
3.  **Переменные оптимизированы?**
    Если при наведении на переменную вы видите `<value optimized out>`, убедитесь, что вы запустили сервер через профиль `Debug` (без флага `--release`), так как оптимизатор LLVM вырезает неиспользуемые локальные переменные в релизных сборках.

## 5. Решение частых проблем при запуске отладки

*   **Ошибка `Address already in use (os error 48)`:**
    Предыдущий запуск сервера не был остановлен корректно и удерживает порты `8090` (TCP-игры) или `8091` (HTTP-админки).
    *Решение:* Найдите и убейте зависший процесс:
    ```bash
    kill -9 $(lsof -t -i:8090)
    ```
*   **Ошибка `M3R_ADMIN_TOKEN is required`:**
    Отсутствует переменная окружения с токеном авторизации.
    *Решение:* При запуске из терминала используйте:
    ```bash
    M3R_ADMIN_TOKEN=local-dev-admin cargo run
    ```
    При запуске из VS Code убедитесь, что вы используете один из преднастроенных лаунч-профилей из `.vscode/launch.json`.
