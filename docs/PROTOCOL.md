# Сетевой протокол

> Это внутренний протокол Mines legacy-клиента и может меняться вместе с клиентской частью.

## Базовый wire-формат

Сетевая шина построена вокруг структуры `server/protocol/mod.rs::Packet`:

- `length: i32 LE` — общий размер кадра в байтах (включая этот префикс).
- `data_type: u8` — тип пакета (`U`, `B`, `J`, ...).
- `event_name: [u8; 2]` — двухсимвольный event.
- `payload: Vec<u8>` — необработанный полезный байтовый буфер.

Запись пакета:
```
[4 bytes length][1 byte data_type][2 bytes event][payload]
```

## Ключевые события (Server → Client)

- `ST` — статусное сообщение (UTF-8 текст).
- `AU` — ответ на авторизацию (ID сессии).
- `PI` — Ping/Pong (формат `pong_resp:client_time:text`).
- `cf` — инициализация мира (JSON с размерами и версией).
- `AH` — хеш авторизации (`user_id_hash`).
- `@T` — телепортация игрока (`x:y`).
- `BI` — информация о боте/другом игроке (JSON).
- `sp` — параметры скорости игрока (`xy_pause:road_pause:depth`).
- `ON` — количество игроков онлайн (`online:max`).
- `P$` — баланс игрока (JSON `{"money":M,"creds":C}`).
- `BD` — состояние авто-копания (`0` или `1`).
- `GE` — координаты игрока (`x:y`).
- `OK` — модальное сообщение (`title#message`).

## Пакет HB (Heartbeat / World Update)
Тип данных `B`, событие `HB`. Содержит набор подпакетов:
- `M` — обновление участка карты.
- `D` — направленный визуальный эффект (FX).
- `C` — чат-баббл над игроком.
- `X` — удаление объекта.
- `F` — эффект взрыва/вспышки.

## Клиентские TY-пакеты (Client → Server)
Действия игрока оборачиваются в `TY`-структуру:
- `event_name: [u8; 4]`
- `time: u32 LE` — клиентский таймстамп.
- `x`, `y: u32 LE` — координаты действия.
- `sub_payload` — параметры действия.

**Типы TY-событий:**
- `XMOV` — движение (payload: ID направления как текст).
- `XDIG` — копание (payload: ID направления).
- `XBLD` — строительство (payload: `{direction}{blockType}`).
- `GUI_` — нажатие кнопки интерфейса (payload: имя кнопки).
- `LOCL` — локальный чат (payload: `length:message`).
- `WHOI` — запрос информации о ботах по списку ID.

## Авторизация (AU)
Клиент отправляет пакет `AU` при подключении. Форматы:
1. `uniq` — серверная авторизация.
2. `uniq_NO_AUTH` — без авторизации.
3. `uniq_userid_token` — стандартная авторизация по ID и токену.

## Обработка и безопасность
- Неверная длина или слишком большие пакеты приводят к немедленному закрытию сессии.
- Имена событий (`event_name`) чувствительны к регистру.
- Все числовые значения передаются в Little-Endian.
