# Orphan-символы pack-спрайтов (задача #5)

Опознание «осиротевших» типов паков `C / B / b / P / Z / c / I` — символов, которые
клиент умеет рендерить, но которых нет в серверном `enum PackType`.

## Метод

Иерархия истины (клиент → C# реф → js реф):

1. **Клиент** `Assets/Scripts/Gameplay/PackSprite.cs::ChangeSpriteToModel()` —
   единственное место, где `_Obj.type` (char) маппится в спрайт из листа
   `Resources/packs`. Это эталон того, что клиент готов нарисовать.
2. **C# enum** `GameShit/Buildings/PackType.cs` — что сервер реально шлёт как
   здание в `O`-оверлее: `T R G M U L F Q O W J Y D` + `' '`(None/Gate). **Точно
   совпадает с Rust `PackType`** (`server/game/buildings.rs:54`).
3. **Единственный не-enum продюсер** — `GameShit/WorldSystem/Chunk.cs:140
   SendPack(char, …)`, вызывается только из `Consumables/ShitClass.cs` тремя
   `SendPack('B', …)`.

Грепы по `server_reference/` и `js_reference/` подтвердили: кроме `'B'` ни один
orphan-символ нигде на провод не пишется.

## Таблица опознания

`num` = индекс в спрайт-листе `packs`. «Аним» = перерисовка в `Update()`.

|-----|--------------|------|-------------|---------------------|---------|
| **B** | 6 (`off1`→36, `off2`→38) | да (пульс `cos·33`) | — | ✅ **C# `ShitClass`** | **НЕ orphan.** Transient item-эффект: `off`=0 Boom, 1 Protector, 2 Razryadka. В Rust уже есть (`heal_inventory.rs:602-613`, паритет). |
| **C** | **−1** | нет | — | ❌ нет | Невидимый/без-спрайтовый пак. `sprites[-1]` бросил бы — значит на провод не шлётся никогда. Sentinel «коллизия без визуала». |

## Вывод

- **`B` исключить из orphan-списка** — это рабочий, реализованный тип (item-эффект),
  попал в список #5 по ошибке.
- **`C b P Z c I` — настоящие сироты:** клиентские спрайт-слоты без серверного
  продюсера ни в C#-, ни в js-референсе. Это либо мёртвые ассеты, либо фичи из более
  поздней версии оригинала, которой у нас нет.
- **Действий на сервере не требуется.** Реализация любого из них (`P` порт, `I`
  иллюзия Левиафана) — это новая фича со своей спекой, а не «портировать
  существующее». Привязаны к задачам #3 (Левиафан → `I`) и потенциально новым.

Факты (num, аним, продюсер) — из кода по `файл:строка`. Назначения `P`/`I` помечены
как **кандидаты** (вывод из спрайта+беклога), не подтверждённый контракт.
