# API и точки интеграции

## Сетевые интерфейсы

### 1. Игровой протокол (TCP)
Единая точка входа для игровых клиентов:
- `0.0.0.0:<port>` (`port` из `config.json`, по умолчанию `8090`).
- Бинарный протокол описан в `docs/PROTOCOL.md`.

## Управляющие файлы

- `config.json` — runtime-конфиг (логирование, порт, размеры мира).
- `cells.json` — описание типов клеток мира (обязателен).
- `buildings.json` — конфигурация построек (ищется в корне или в `data/`).
- `openmines.db` — SQLite-база данных. При старте сервер автоматически мигрирует старый файл `mines3.db` в `openmines.db`, если он существует.
- `*.mapb` — бинарные дампы чанков мира в директории состояния.

## Переменные окружения (M3R_*)

- `M3R_DATA_DIR` — путь к директории для файлов состояния (БД, карты). По умолчанию — корень или `artifacts/runtime`.
- `M3R_REGEN_WORLD=1` — полная регенерация мира при старте (удаляет `.mapb` и постройки из БД).
- `M3R_GRANT_ADMIN=name1,name2` — автоматически выдать права администратора указанным игрокам.

## Аргументы запуска

- `--regen` / `--regen-world` — аналогично `M3R_REGEN_WORLD`.
