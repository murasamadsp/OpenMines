# API и точки интеграции

## Сетевые интерфейсы

### 1. Игровой протокол (TCP)
Единая точка входа для игровых клиентов:
- `0.0.0.0:<port>` (`port` из `config.json`, по умолчанию `8090`).
- Бинарный протокол описан в `docs/PROTOCOL.md`.

### 2. Ops HTTP API (localhost)
Служебный API для мониторинга и диагностики. Активируется через `M3R_OPS_HTTP=1`.
По умолчанию слушает `127.0.0.1:8092`.

**Эндпоинты:**
- `GET /healthz` — проверка жизнеспособности.
- `GET /readyz` — проверка готовности.
- `GET /metrics` — метрики в формате Prometheus.
- `GET /ops/world/cell?x=..&y=..` — детальная информация о клетке.
- `GET /ops/packs/at?x=..&y=..` — информация о постройке (pack) в координатах.
- `GET /ops/world/reseed_durability?x=..&y=..` — сброс прочности клетки к значениям из `cells.json`.

### 3. Mapviewer HTTP API
API для браузерного просмотрщика карт.
- `GET /api/map/meta` — метаданные мира (размеры, имя).
- `GET /api/map/chunk/:cx/:cy` — сырые данные чанка (бинарный поток клеток).

## Управляющие файлы

- `config.json` — runtime-конфиг (логирование, порт, размеры мира).
- `cells.json` — описание типов клеток мира (обязателен).
- `buildings.json` — конфигурация построек (ищется в корне или в `data/`).
- `openmines.db` — SQLite-база данных. При старте сервер автоматически мигрирует старый файл `mines3.db` в `openmines.db`, если он существует.
- `*.mapb` — бинарные дампы чанков мира в директории состояния.

## Переменные окружения (M3R_*)

- `M3R_DATA_DIR` — путь к директории для файлов состояния (БД, карты). По умолчанию — корень или `artifacts/runtime`.
- `M3R_REGEN_WORLD=1` — полная регенерация мира при старте (удаляет `.mapb` и постройки из БД).
- `M3R_GRANT_ADMIN=name1,name2` — автоматически выдать права администратора указанным игрокам.
- `M3R_OPS_HTTP=1` — включить Ops-интерфейс.
- `M3R_OPS_HTTP_PORT=8092` — порт для Ops API.
- `M3R_OPS_HTTP_BIND_PUBLIC=1` — разрешить доступ к Ops API не только с localhost.
- `M3R_MAPVIEWER_TOKEN=...` — токен авторизации для Mapviewer (Bearer / X-Map-Token).

## Аргументы запуска

- `--regen` / `--regen-world` — аналогично `M3R_REGEN_WORLD`.
