# API и точки интеграции

## Что есть из API

На текущий момент у сервера нет REST/GraphQL API.  
Единая внешняя точка интеграции — TCP-сокет с бинарным игровым протоколом:

- `0.0.0.0:<port>` (`port` из `config.json`, по умолчанию `8090`)
- wire-фрейм описан в `docs/PROTOCOL.md`

## Управляющие файлы

- `config.json` — runtime-конфиг (логирование, порт, размеры мира).
- `cells.json` — описание клеток мира.
- `openmines.db` — локальная SQLite-база (создаётся при первом запуске; при обновлении со старых сборок поддерживается переименование с `mines3.db`).
- `*.mapb` — кэшированные файлы мира в корне запуска.

## Аргументы запуска

В `server/main.rs` доступны дополнительные флаги:

- `--regen` / `--regen-world` — удалить `*.mapb` перед стартом.
- `M3R_REGEN_WORLD=1` — альтернативная активация очистки мира через переменную окружения.

## Как потреблять данные извне

Если нужно автоматическое развертывание:

- используйте Docker image (`ghcr.io/<owner>/openmines-server:<tag>`);
- для операций с БД и логами — это файловые артефакты внутри контейнера (`/data`).

## Совместимость

Если меняете клиентский протокол или логику net-слоя, обязательны:

1. обновление обработчиков в Unity-скриптах,
2. обновление документации `docs/PROTOCOL.md`,
3. описание миграции в `CHANGELOG.md`.
